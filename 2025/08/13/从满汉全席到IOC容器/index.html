从满汉全席到IOC容器

发表于 2025-08-13 更新于 2025-08-14
本文字数： 3.9k 阅读时长 ≈ 4 分钟

记录下自己对于IOC容器的理解。

在跟着网上的项目时，我经常看到@AIService可以自动调用放入到IOC容器中的对象。以此产生了多个问题：什么是IOC容器，为什么@AIService可以自动调用对象，@Bean又是什么？

1.第一个问题，什么是IOC容器：

IOC是控制反转的意思，英文Inversion of Control。意思是原本一个创建的对象由程序员控制变成让系统控制。现在让外部容器来负责对象的创建和依赖关系的管理。通过移交控制权，我们不需要自己管理和创建对象。这一优势在对象互相依赖的情况下会表现得特别明显。更官方地来说：

IOC容器是实现控制反转的具体工具，它负责：

  • 对象创建：根据配置自动创建对象实例
  • 依赖注入：自动将依赖的对象注入到需要的地方
  • 生命周期管理：管理对象从创建到销毁的整个过程
  • 配置管理：通过配置文件或注解来定义对象间的关系

进一步的，我们可以从代码来理解：

// 传统方式 - 对象自己控制依赖
class UserService {
    private UserRepository userRepo;
    private EmailService emailService;
    
    public UserService() {
        this.userRepo = new DatabaseUserRepository(); // 硬编码依赖
        this.emailService = new SMTPEmailService();   // 硬编码依赖
    }
}

// IOC方式 - 容器控制依赖
class UserService {
    private UserRepository userRepo;
    private EmailService emailService;
    
    // 依赖通过构造函数注入，不再自己创建，相当于写了一堆接口，这个过程就是写菜单
    public UserService(UserRepository userRepo, EmailService emailService) {
        this.userRepo = userRepo;
        this.emailService = emailService;
    }
}

如果我们把IOC容器当作一个菜单，每一个对象就是里面的一道菜，这里的每道菜就是通过@Bean注入的对象。我们只需要先确认客户的需求（下订单），然后提供实现功能的方法（菜谱），最后交给厨师（dev.langchain4j.service.spring.AiService）。这样，程序就会根据dev.langchain4j.service.spring.AiService包里写好的方法自动组合这些方法，实现完整的功能。

这同样也是开发的流程：1找到合适的框架 2学会框架的游戏规则 3组合框架解决问题 4专注于业务逻辑

2.@AiService 这种格式是什么

这种以 @ 符号开头的单词，在 Java 编程语言中有一个正式的名称，叫做 注解 (Annotation)。核心作用是为代码（比如类、方法、变量）添加额外的信息（元数据），但它本身并不直接改变代码的执行逻辑。

这种做法是Java 语言自己的语法规则，当我们启动一个Spring 应用时，他会运行一个反射机制，让一个程序在运行的时候去检查和操作自己内部的结构（比如查看一个类有哪些方法、哪些字段、以及这些字段上贴了哪些注解）。

具体来说Spring 的工作流程是这样的：

回到注解这个东西上，这里我总结了下我遇到的几个：

  1. Spring框架核心注解
  • @Configuration ：标记 CommonConfig 类为配置类，Spring会扫描并加载其中的Bean定义。
  • @Autowired ：自动装配Spring容器中的Bean，如在 CommonConfig 中注入 chatMemoryStore 、 embeddingModel 等，在 ChatController 中注入 consultantService 。
  • @Bean ：声明一个Bean，方法返回值将被注册为Spring容器中的Bean，如 chatMemoryProvider() 、 Store() 、 contentRetriever() 等方法。
  • @RestController ：组合注解，标记 ChatController 类为REST控制器，处理HTTP请求。
  • @RequestMapping ：指定请求路径和生产的媒体类型，如 /chat 接口配置。
  1. langchain4j框架AI服务注解
  • @AiService ：langchain4j框架的核心注解，用于标记 ConsultantService 接口为AI服务接口，支持自动创建代理对象。
    • wiringMode ：指定装配模式（ EXPLICIT 表示手动装配）。
    • streamingChatModel ：指定流式聊天模型的Bean名称（ openAiStreamingChatModel ）。
    • chatMemoryProvider ：指定会话记忆提供者的Bean名称（ chatMemoryProvider ）。
    • contentRetriever ：指定内容检索器的Bean名称（ contentRetriever ）。
    • tools ：指定工具类的Bean名称（ reservationTool ）。
  • @SystemMessage ：指定系统消息，可以从资源文件加载（如 fromResource = “system.txt” ）。
  • @MemoryId ：标记参数为记忆ID，用于标识不同的会话。
  • @UserMessage ：标记参数为用户消息，作为AI服务的输入。

3.如何拓展程序功能

现在我们知道了JAVA里面的注释是做什么的，知道了IOC容器是什么。明确了开发流程。现在我们要开始了解一个程序应该如何进行功能上的扩展。

结合之前的大模型项目，这个扩展应该分为两种，一类是langchain4j框架支持的AI扩展服务，比如说对于流式传输的支持，对于数据存储的支持。另一种是扩展工具，这类工具就不是langchain4j框架支持的AI服务，而是更大的一个框架sping里面的基础设施。

对于第一类服务，我们需要先定义接口，这里会使用 @AiService 注解标记，在这里写上需要使用的接口名字，再去对应的写出方法并用@Bean注入到IOC容器中就行。

对于第二种服务，我们需要先创建一个普通的Java类，如 ReservationTool 。标记为Spring组件，并使用 @Component 注解将类标记为Spring组件，使其被Spring容器管理。

但和第一种方法，只需要写接口和方法不一样，我们还需要写更多的执行逻辑。总结下来：

核心流程总结

  1. 第一类服务（AI服务） ：
  • 定义接口 → 配置AI组件 → 注入Spring容器
  1. 第二类服务（工具类） ：
  • 实现业务逻辑 → 标记为Spring组件 → 标记为AI工具 → 被AI服务引用
