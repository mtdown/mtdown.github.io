总结学习maven

发表于 2025-08-13 更新于 2025-08-20
本文字数： 795 阅读时长 ≈ 1 分钟

因为我之前是使用python的多，这里说下我对maven学习后的一些理解。先下定义：maven的功能是编译java代码，在依赖管理方面使用pom.xml文件。不过和conda不同，maven管理环境依赖的方式是把需要的内容下载到一个本地仓库（.m2/repository），然后根据pom.xml里面的需求调用需要的内容。

最后，我们可以归纳他们的特性为：

特性 Maven Conda
存储位置 中央化本地仓库 (~/.m2/repository) 隔离环境目录 (<conda_root>/envs/<env_name>/)
依赖共享 所有项目共享同一个仓库 环境间依赖完全隔离 (独立副本)
隔离单位 项目级别 (通过 Classpath 控制) 环境级别 (物理目录和系统路径隔离)
隔离机制 逻辑隔离 (依赖版本选择 + Classpath) 物理隔离 (文件系统副本 + PATH 环境变量)
包含内容 仅依赖库 (JAR) 及其元数据 Python解释器、Python包、可执行文件、非Python库
依赖问题 项目运行时依赖指向中央仓库的JAR。 环境目录本身就是一个包含所有依赖的包
