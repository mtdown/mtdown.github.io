langchain4j项目总结以及改进计划-从localhost到公网

发表于 2025-08-11 更新于 2025-08-18 阅读次数：
本文字数： 1.2k 阅读时长 ≈ 1 分钟

在之前的任务中，我完成了大语言模型项目的实现，但是仍有不少问题：1.大模型只能访问本机，使用localhost，我希望能够在公网访问。2.大模型的数据库，我希望是使用我的博文进行回答。3.我的大语言模型最好还能有联网功能，这样就能得到更准确的回答。

1.从localhost到公网

问题和处境：

我有一个在本地 Windows 电脑上运行的大模型应用（跑在 localhost:8080），想把它发布到我的个人域名 mtdown.top 上，但面临几个棘手问题：

  1. 域名 mtdown.top 已经托管了我的 Hexo 博客，不能影响它。
  2. 我没有公网服务器，也不想为此花钱，就打算使用我自己的电脑，没有公网 IP。

因此我计划使用Cloudflare Tunnel实现这个目的,利用子域名ai.mtdown.top给AI 项目使用。不过首先，我需要完成域名的迁移。

1.1域名转移

1.第一件事是去注册一个Cloudflare账号，在注册完成后，你会看到一个这样的界面，按顺序输入你的域名然后确定就行，需要注意的是，你的域名不需要加https：,或者是www这些，只输入最核心的部分，我的就是mtdown.top

![image-20250818190121531](langchain4j项目总结以及改进计划/image-20250818190121531.png

2.然后你会看到一个资费界面，这时候选择free那一档就可以，接下来他会自动扫描你的之前的域名解析的DNS。

3.因为我之前在阿里云买的域名，在那里解析了DNS，所以需要去阿里云后台修改 DNS 服务器地址，让Cloudflare 接管我的域名。关于域名修改这里，Cloudflare 会给出修改后的域名应该是什么，按照如何修改域名的DNS_域名(Domain)-阿里云帮助中心这里的方法去修改就行了

至此，第一阶段完成，现在是Cloudflare管理我的域名了

1.2创建并配置 Cloudflare Tunnel

为了让我本地的项目可以在网络上看到，我需要在我的 Windows 电脑和 Cloudflare 网络之间，建立一条安全的加密隧道。

选择Cloudflared 即可。点击 Next。

输入名字。点击 Save tunnel。

接下来就是按照他给的信息按顺序操作123，最后以管理员权限打开任务管理器运行第四条指令，一切就都大功告成啦！
