1.2 IOC控制了什么，DI是什么？

IOC的作用就是控制反转，从程序员创造一个对象到系统自己创造一个对象。IOC的提出是为了解决后期维护问题，降低代码和代码之间的耦合性。DI指的是依赖注入。

1.3 早期的Spring的DI和现在的DI

视频中，作者展示了一种注入方法，使用 和 标签将对象注入。不过这种是比较早期的方法了（Spring1），现在我们是使用@Bean来进行注入。

更进一步的(Spring2)，就是用@Component结合Spring.xml配置文件

剩下的变化没有一一列举，只给出了这样的表格：

版本 版本特性
Spring Framework 1.0 1. 所有代码都在一个项目中
2. 支持核心功能 IoC、AOP
3. 内置支持 Hibernate、iBatis 等第三方框架
4. 对第三方技术简单封装，如：JDBC、Mail、事务等 5. 只支持 XML 配置方式
Spring Framework 2.0 1. 最重要的变化：支持注解配置
2. 增加对 ehcache 支持
3. 对 JMS (Java消息服务)、JMX (Java管理扩展) 等支持
Spring Framework 3.0 1. 支持 Java 类的配置
2. 由原来的一个项目拆分成多个子模块
3. 添加单元测试子模块
Spring Framework 4.0 1. 全面支持 Java 8
2. 添加二级子项目 Spring MVC，基于 Servlet 3.0 开发
3. 提供 WebSocket 支持 4. 引入 @RestController 等注解，增强对 Restful 的支持
Spring Framework 5.0 1. 最低环境要求 Java 8
2. 日志整合：把原来 commons-logging 封装后变为 spring-jcl 模块
3. 跨域更改：把原来的 JSONP 支持变更为修改响应头的 @CrossOrigin 注解
4. 响应式编程：添加二级子项目 WebFlux
5. 使用 Kotlin (科特林) 进行函数式编程
Spring Framework 6.0 1. 最低环境要求 Java 17
2. 加强对响应式编程支持：添加子模块 spring-r2dbc
3. 取消子模块 spring-orm-hibernate4，只保留 spring-orm，只支持 Hibernate5。

1.4@Component，@Bean，@Import相关

在Spring框架中最核心的注入方式有两种，一种是@Bean注入，另一种是@Component注入。其中，@Component有很多子注解，比如说@Service, @Repository, @Controller 等。这些都是属于功能更加细致化的产物.

特性 @Component @Bean
位置 写在类上面 写在方法上面你
控制类型 业务逻辑控制 对象创建（实例化）控制
控制内容 控制一个类能做什么，其内部方法的具体实现。 控制一个对象如何诞生，包括调用哪个构造函数、设置哪些属性、以及根据条件返回不同类型的实例
强大之处 对内部实现的100%掌控。 对创建过程的100%掌控。

1.5 @Bean的示例化

没有指定，使用默认构造器实例化的情况：

  1. 有多个构造函数时，使用无参构造器，如果没有无参构造器，就会报错
  2. 如果只有一个构造函数，@Bean会自动实例化唯一的哪一个；

使用实例工厂的情况：

  1. 可以自由选择构造函数进行实例化

使用FactoryBean的情况：

  1. FactoryBean是一个接口
  2. 前提是有一个Bean，一旦Bean实现了FactoryBean，他就成为了一个特殊的Bean
  3. 同样的，这种做法也可以自由选择构造函数
