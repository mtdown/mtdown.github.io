文件和目录管理

1.ls系列命令：

​ ls：列出目录内容。

​ ls -l：以长格式显示（包括权限、所有者、大小、修改日期等）：这里面的第一列是权限相关；如果第一列的第一个字母以d开头说明是目录，l开头说明是符号连接，-开头说明是普通文件。之后每三位一组rwx = 读/写/执行，分别是所有者权限，所属组权限和其他用户权限。这就是第一列的全部含义。

​ 第二列的数字对于目录d，表示的是子目录的数量。比如你打开Agent这个目录，下面就会有3个子文件。对于文件-，表示的是指向该文件的硬链接综述。

​ 第三列第四列分别表示用户名和文件所在的组名；

​ 第五列表示的是文件大小，不过对于目录d来说表示的是目录元数据占用的磁盘空间，只有-类型的文件才是真实的大小

​ 第六列到第八列表示的是最后修改的时间，然后就是第九列的文件名。这就是对于ls -l看到的内容的全部解释。

​ ls -a：显示所有文件，包括隐藏文件（以.开头的文件）但是它显示的内容只有文件夹的名字。

​ ls -la：以ls -l的格式显示所有文件，包括隐藏文件（以.开头的文件）。太多了就不完整截图了

​ ls -lh：以人类可读的格式显示文件大小（例如，1K, 234M, 2G）。其实就是把第四列换成了kb,Mb这样的单位，而不是默认的b。

​ 更进一步的，我们还有ls -lah这样显示全部文件并以kb,Mb格式展示的组合，当然ls -lha这样替换也是OK的。a:展示隐藏，l：格式化输出，h：以人类可读的格式显示文件大小。此外，我们还有缩写ll命令，可以直接显示出ls -lah的结果

2.切换目录

cd /path/to/directory：进入指定目录。这里的【/path/to/directory】替换成你想去的目录即可，这里可以输入绝对路径，如果要进入当前目录的子目录，比如你当前的目录是data，要进入子目录math里面的highschool目录,【/path/to/directory】替换成【math/highschool】，最后的命令也就是cd math/highschool。

cd ..：返回上一级目录。你甚至可以继续叠加cd ../..进入上层目录的上层目录;然后win系统下面是\

cd ~ 或 cd：返回当前用户的主目录。

cd -：返回上次所在的目录

pwd (print working directory)：显示当前工作目录的路径。

3.目录的增删改查

mkdir (make directory)：创建新目录。

  1. mkdir my_new_folder：创建一个名为 my_new_folder 的目录。
  2. mkdir -p a/b/c：创建多级目录，如果父目录不存在也会一并创建。

rmdir empty_folder：删除名为 empty_folder 的空目录。

cp (copy)：复制文件或目录。这里面的-r是比较通用的，对于批命令基本都需要用到-r

  1. cp file.txt new_file.txt：复制文件 file.txt 并命名为 new_file.txt。
  2. cp -r folder/ new_folder/：递归复制目录及其内容。

mv (move)：移动或重命名文件/目录。

  1. mv file.txt /path/to/new_location/：将 file.txt 移动到新位置。
  2. mv old_name.txt new_name.txt：将 old_name.txt 重命名为 new_name.txt。

rm (remove)：删除文件或目录。经典高危指令，一定一定是和上面的cp先保存个备份再使用。不然很容易出大问题

  1. rm file.txt：删除文件 file.txt。
  2. rm -r folder/：递归删除目录及其内容。
  3. rm -rf folder/：强制递归删除目录及其内容，不提示确认。

5.权限管理

chmod (change mode)：改变文件或目录的权限。

  1. chmod +x script.sh：给脚本添加执行权限。
  2. chmod 755 file.txt：将文件权限设置为 rwxr-xr-x。
  • 权限数字解释：r=4, w=2, x=1。
  • 7 = 4+2+1 (rwx) -> 读、写、执行
  • 6 = 4+2 (rw-) -> 读、写
  • 5 = 4+1 (r-x) -> 读、执行
  • 4 = 4 (r–) -> 读
  • 第一个数字代表文件所有者权限，第二个代表同组用户权限，第三个代表其他用户权限。

chown (change owner)：改变文件或目录的所有者。
chgrp (change group)：改变文件或目录的所属组。

6.快捷键：

这些才是真的用得特别多：

  • Tab：命令和文件名自动补全，按两次 Tab 可以显示所有可能选项。
  • Ctrl + C：终止当前运行的命令。
  • Ctrl + D：输入文件结束符（EOF），通常用于退出shell。
  • Ctrl + L：清屏，相当于 clear 命令。
  • Ctrl + R：反向搜索历史命令。
  • 上下箭头：浏览历史命令。

7.vim命令

Vim 有三种常用模式：普通模式 (Normal Mode)、插入模式 (Insert Mode) 和 **命令模式 (Command Mode)**。打开文件后默认进入普通模式。

### 进入/退出

  • vim 文件名：打开或创建一个文件。
  • :w：保存文件。
  • :q：退出Vim。
  • :wq或:x：保存并退出。
  • :q!：强制退出，不保存修改。

### 普通模式 (移动与编辑)

  • i：在光标前进入插入模式。
  • a：在光标后进入插入模式。
  • o：在当前行下方新建一行并进入插入模式。
  • Esc：从插入模式返回普通模式。
  • h, j, k, l：分别向左、下、上、右移动光标。
  • gg：移动到文件第一行。
  • G：移动到文件最后一行。
  • dd：删除当前行。
  • yy：复制当前行。
  • p：在光标后粘贴。
  • u：撤销上一步操作。
  • Ctrl + r：重做（反撤销）。
  • x：删除光标所在的字符。

### 命令模式 (搜索与替换)

  • /关键字：从上往下搜索关键字，按 n 查找下一个。
  • ?关键字：从下往上搜索关键字，按 n 查找下一个。
  • :%s/旧内容/新内容/g：全文替换所有“旧内容”为“新内容”。

8.nano命令

  • nano 文件名：打开或创建一个文件。
  • Ctrl + O：保存文件（Write Out）。会提示确认文件名，直接按回车即可。
  • Ctrl + X：退出 Nano。如果文件有未保存的修改，会提示是否保存。
  • Ctrl + K：剪切当前行（可用于删除行）。
  • Ctrl + U：粘贴被剪切的内容。
  • Ctrl + W：搜索关键字（Where is）。
  • Ctrl + \：替换关键字或文本。
  • Ctrl + G：显示帮助文档。
  • Alt + U：撤销上一步操作。
  • Alt + E：重做（反撤-销）。

一些杂谈：

nohup 命令，用于“让指令在关闭 SSH 后也保持运行”的最经典、最基础的工具。它的名字 nohup 就是 “no hang up” (不要挂断) 的缩写。

nohup 的工作原理 (为什么它是通用的)

  1. 当您通过 SSH 连接到服务器时，您开启了一个会话 (session)。
  2. 当您关闭 SSH 窗口时，系统会向这个会话里所有正在运行的程序发送一个“挂断”信号 (SIGHUP)。
  3. 程序收到这个信号后，默认的反应就是终止自己。
  4. nohup 命令的作用，就是在您的指令前加一个“保护罩”。它会告诉您的指令：“请忽略那个挂断信号，继续运行你的任务”。
  5. 为了确保指令能在后台运行，请一定要将 nohup 和 & 符号结合使用。

类似的，还有 screen ，可随时查看实时进度条

screen 是一个“终端多路复用器”，可以把它想象成一个可以在服务器上持久运行的虚拟终端窗口。你把任务放进这个窗口里，即使断开了SSH，这个窗口和里面的任务也依然在运行

  1. 检查并安装 screen

  2. 创建一个新的 screen 会话并进入** 我们用 -S 参数给这个会话起一个名字，比如 download_task，方便以后找回。

  3. 在 screen 会话中，运行你的下载命令

  4. “分离”会话，让它在后台运行：先按 Ctrl + A，然后松开，再按 D就能返回到原来的主终端界面
