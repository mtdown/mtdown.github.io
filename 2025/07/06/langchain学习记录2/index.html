1.2 ConsultantService文件

决定了依赖，接下来就是要定义接口，也就是ConsultantService文件。其在整个项目中，负责定义AI服务，主要是通过 @AiService 注解来将多种AI功能组件整合在一起。这里重点说下里面wiringMode，它默认为 AiServiceWiringMode.AUTOMATIC ，会自己去装配 IOC 容器中已有的 chatModel。

此外，对于langchain4j支持的东西，他们都会提供专用的属性来配置：比如：chatModel / streamingChatModel：配置AI的“大脑”，即语言模型本身；chatMemoryProvider：配置AI的“内存”，负责管理对话历史。；contentRetriever：配置AI的“知识库检索器”，负责RAG中的信息检索。

而对于自定义的外部功能，则是通过tools属性来实现，作为AI的外部工具箱。@AiService里面的对象名字是@Bean注入的容器名字。

1.3 CommonConfig.java文件

这个文件是用来配置项目需求运行所必须的内容的，当我们要添加一个新功能时，往往是先去找能实现对应功能的包，然后构建一个Bean对象出来。统一管理后，就是这个 CommonConfig.java文件

1.3 application.yml文件

application.yml文件是Spring Boot 应用的主配置文件，包含应用程序的各种配置项，如服务器端口、数据库连接信息、日志级别、自定义属性等。每要新接入一个组件，都需要在这个文件中写入对应的设置，比如说使用的模型名字啊，端口号这些。

此外，根据情况不同，还可以按照application-{profile}.yml 格式命名使用多个配置文件: - application.yml (默认配置) - application-dev.yml (开发环境) - application-prod.yml (生产环境)

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication app = new SpringApplication(Application.class);
        
        // 设置配置文件名
        app.setAdditionalProfiles("dev");
        
        // 设置配置文件位置
        System.setProperty("spring.config.location", "classpath:/custom/");
        
        app.run(args);
    }
}

1.4 tips

像是`public ChatMemoryProvider chatMemoryProvider() {//构建会话记忆对象

​ ChatMemoryProvider chatMemoryProvider = new ChatMemoryProvider() 这里面的这种重写，叫做匿名内部类特性，适用于接口、抽象类和非final`的普通类：

1.实现一个接口：也就是当原本的类是一个inference时：

语法：new InterfaceName() { ... }

含义：这不是对接口的实例化，而是创建一个无名的新类，并且必须立即在大括号 {...} 中提供接口所有抽象方法的具体实现。

2.继承一个抽象类

语法：new AbstractClassName() { ... }

含义：创建一个继承了该抽象类的新类，同样必须立即在大括号 {...} 中提供所有抽象方法的具体实现，非抽象方法选择性重写。

3.继承一个普通的具体类

语法：new ConcreteClassName() { ... }

含义：您在创建一个继承了该具体类的新类，并且可以在大括号 {...} 中重写父类中已有的方法，为其提供一个一次性的、特殊的行为。
