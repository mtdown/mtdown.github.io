1.2 ConsultantService文件

决定了依赖，接下来就是要定义接口，也就是ConsultantService文件。其在整个项目中，负责定义AI服务，主要是通过 @AiService 注解来将多种AI功能组件整合在一起。这里重点说下里面wiringMode，它默认为 AiServiceWiringMode.AUTOMATIC ，会自己去装配 IOC 容器中已有的 chatModel。

1.3 application.yml文件

application.yml文件是Spring Boot 应用的主配置文件，包含应用程序的各种配置项，如服务器端口、数据库连接信息、日志级别、自定义属性等。每要新接入一个组件，都需要在这个文件中写入对应的设置，比如说使用的模型名字啊，端口号这些。

此外，根据情况不同，还可以按照application-{profile}.yml 格式命名使用多个配置文件: - application.yml (默认配置) - application-dev.yml (开发环境) - application-prod.yml (生产环境)

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication app = new SpringApplication(Application.class);
        
        // 设置配置文件名
        app.setAdditionalProfiles("dev");
        
        // 设置配置文件位置
        System.setProperty("spring.config.location", "classpath:/custom/");
        
        app.run(args);
    }
}
