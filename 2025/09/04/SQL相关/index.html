1.什么是CRUD

CRUD 是一个缩写，代表了在处理持久化存储（最常见的就是数据库）时，四种最基本、最核心的操作。也就是所谓的增删改查,这是一个统称，不止是用在mysql，web服务器里也是有的。

英文 功能 SQL命令 Web 服务
Create 创建数据 Insert POST
Delete 删除数据 delete DELETE
Update 修改数据 update PUT或PATCH
Read 查找数据 select GET

2.join的方式

1. 内连接 (INNER JOIN)

这是最常用的一种JOIN，它只返回两个表中连接字段相匹配的行。可以理解为取两个表的“交集”。实际使用上好像就写个JOIN就行

2.左连接 (LEFT JOIN / LEFT OUTER JOIN)

以左边的表（FROM后面的第一个表）为基础，返回左表的所有行。对于右表中没有匹配记录的行，结果集中右表的列会显示为 NULL。

3. 全外连接 (FULL OUTER JOIN)

这是左连接和右连接的并集。它会返回两个表中的所有行。如果某一行在另一个表中没有匹配，那么另一个表的列将显示为 NULL。

4.自然连接 (NATURAL JOIN)

它是一个简化的内连接，SQL会自动寻找两个表中所有同名且数据类型相同的列作为连接条件，然后进行内连接。

5交叉连接 (CROSS JOIN)

也称为笛卡尔积。它会返回左表中的每一行与右表中每一行的所有可能组合。结果行数等于 左表行数 × 右表行数。

总结

JOIN类型 关键字 描述
内连接 INNER JOIN 只返回两个表中能匹配上的行（交集）。
左连接 LEFT JOIN 返回左表的所有行，以及右表中匹配的行。
右连接 RIGHT JOIN 返回右表的所有行，以及左表中匹配的行。
全外连接 FULL OUTER JOIN 返回左表和右表中的所有行（并集）。
自然连接 NATURAL JOIN 自动以内连接方式合并所有同名的列。
交叉连接 CROSS JOIN 返回两个表的笛卡尔积，即所有可能的组合。

4.数据库里面的运算符

标准的比较运算符 (=, >, <, <>) 只能用于一个值对一个值的比较。当你的子查询有可能返回超过一行的结果时，直接使用这些比较符就会导致错误。这在SQL中被称为“Subquery returns more than 1 row”的经典错误。

在这个例子中，这个比较方法就会面对一个问题：单个salary要和多个集合比较

总结一下情况：

一。当子查询只返回单个值时：

  1. 比如子查询使用了MAX(), MIN(), AVG()等聚合函數（且沒有GROUP BY）。

  2. 子查询的WHERE条件是基于主键的精确查找

    在这个情况下，可以使用=,>,<,>=,<=等

二。当子查询返回一列值或者集合时：

使用IN,NOT IN,ANY,SOME,ALL等

类别 运算符 通俗解释 (“读作”) 核心使用场景
单值比较 =,>,<,>=,<= 一个值 对比 一个值 子查询确保只返回一个标量值时使用。
集合成员 IN / NOT IN 值是否 存在于 / 不存在于 列表之中？ 检查一个值是否是子查询返回的多个结果中的一员。
集合比较 > ANY 或 > SOME 是否比列表中的 至少一个 大？ (即 > 最小值) 与集合中的“最低门槛”比较。
集合比较 < ANY 或 < SOME 是否比列表中的 至少一个 小？ (即 < 最大值) 与集合中的“最高门槛”比较。
集合比较 > ALL 是否比列表中的 每一个 都大？ (即 > 最大值) 超越集合中的“最高标准”。
集合比较 < ALL 是否比列表中的 每一个 都小？ (即 < 最小值) 低于集合中的“最低标准”。
集合比较 NOT EXISTS 不存在于列表中 通常性能最好，利用索引且有短路特性。对NULL免疫。IN和NOT IN在这方面有问题

5.MINMAX函数适用范围

适用范围：任何可以被排序 (ORDER BY) 的数据类型，都可以使用 MIN() 和 MAX()

根据这条黄金法则，我们来看看MIN()和MAX()都能用在哪些具体的数据类型上：

1. 所有数值类型 (Numeric Types)

  • 类型： INT, BIGINT, DECIMAL, FLOAT, DOUBLE 等。
  • 排序逻辑： 按数字大小排序。
  • 示例： MIN(salary) 返回最低薪水，MAX(age) 返回最大年龄。这是最直观的用法。

2. 所有日期和时间类型 (Date and Time Types)

  • 类型： DATE, DATETIME, TIMESTAMP, TIME。
  • 排序逻辑： 按时间先后顺序排序。更早的日期/时间被认为是“更小”的。
  • 示例： MIN(event_date) 返回“最早”的日期，MAX(login_time) 返回“最晚”的登录时间。这完美地解答了你上一题的疑惑。

3. 所有字符串类型 (String Types)

  • 类型： CHAR, VARCHAR, TEXT。
  • 排序逻辑： 按字典顺序 (Lexicographical Order) 排序。这个顺序取决于数据库的字符集和排序规则（Collation）。
  • 示例：
    • 对于 {'a', 'c', 'b'} 这一组值，MIN() 返回 'a'，MAX() 返回 'c'。
    • 对于 {'Apple', 'Banana', 'apple'} 这一组值，MIN() 通常返回 'Apple'（大写字母在前），MAX() 返回 'apple'。
  • 一个重要的陷阱： 当数字被存为字符串时，排序结果可能出乎你的意料。
    • 对于字符串 {'10', '2', '100'}，MIN() 返回的是 **'10'**，MAX() 返回的是 **'2'**！因为它是一位一位地按字典顺序比较，'1' 开头的比'2'开头的要“小”。

4. 枚举类型 (ENUM)

  • 类型： ENUM('small', 'medium', 'large')。
  • 排序逻辑： 按你在定义ENUM类型时列出的值的先后顺序排序。
  • 示例： 对于上面的ENUM，MIN()会返回'small'，MAX()会返回'large'。

5.不适用的类型

根据黄金法则，那些不具有天然排序逻辑的数据类型，就无法使用MIN()和MAX()。

  • 二进制/大对象类型 (Binary/BLOB Types):
    • 类型： BLOB, BINARY, VARBINARY 等。
    • 原因： 这些类型通常用来存储图片、文件等二进制数据。数据库不知道一张图片和另一张图片哪个“更大”或“更小”，它们没有可比较性。

结论：“能排序，就能取最值。”

6.SQL子句分类

DQL (Data Query Language) - 数据查询语言

  • 核心: SELECT
  • 作用: 用于从数据库中查询和检索数据，是你目前接触最多的部分。它不改变数据库中的任何数据，只是读取。

DML (Data Manipulation Language) - 数据操作语言

  • 核心: INSERT, UPDATE, DELETE
  • 作用: 用于操作（增、删、改）数据库表中的数据。
    • INSERT: 插入新数据行。
    • UPDATE: 更新已存在的数据行。
    • DELETE: 删除数据行。

DDL (Data Definition Language) - 数据定义语言

  • 核心: CREATE, ALTER, DROP, TRUNCATE
  • 作用: 用于定义和管理数据库的结构（模式，Schema）。它操作的对象是表、索引、视图等数据库对象本身，而不是里面的数据。
    • CREATE: 创建新的数据库、表、视图等。
    • ALTER: 修改已存在的数据库对象的结构。
    • DROP: 完全删除一个数据库对象。

DCL (Data Control Language) - 数据控制语言

  • 核心: GRANT, REVOKE
  • 作用: 用于管理数据库的访问权限和安全级别。
    • GRANT: 授予用户或角色权限。
    • REVOKE: 撤销用户或角色的权限。

TCL (Transaction Control Language) - 事务控制语言

  • 核心: COMMIT, ROLLBACK, SAVEPOINT
  • 作用: 用于管理事务，确保数据操作的原子性和一致性。
    • COMMIT: 提交事务，将事务所做的所有修改永久保存。
    • ROLLBACK: 回滚事务，撤销当前事务中所有未提交的修改。

7.SQL常用函数记录

1.聚合函数 (Aggregate Functions)

聚合函数对一组值进行计算，并返回一个单一的值。它们通常与 GROUP BY 子句一起使用，用于数据统计和汇总。

函数 描述 示例
COUNT() 计算行数。COUNT(*) 计算所有行，COUNT(column) 计算该列非NULL值的行数。 SELECT COUNT(*) FROM Users; (统计用户总数)
SUM() 计算数值列的总和。 SELECT SUM(price) FROM Orders; (计算订单总金额)
AVG() 计算数值列的平均值。 SELECT AVG(score) FROM Grades; (计算平均分)
MAX() 找出列中的最大值。 SELECT MAX(salary) FROM Employees; (查找最高工资)
MIN() 找出列中的最小值。 SELECT MIN(birth_date) FROM Users; (查找最早的出生日期)

2.字符串函数 (String Functions)

函数 描述 示例
CONCAT() 连接两个或多个字符串。 SELECT CONCAT(first_name, ‘ ‘, last_name) FROM Users; (拼接姓名)
LENGTH() / CHAR_LENGTH() 获取字符串的长度。LENGTH 是字节长度，CHAR_LENGTH 是字符长度。 SELECT CHAR_LENGTH(‘你好世界’); (返回 4)
UPPER() / LOWER() 将字符串转换为大写或小写。 SELECT UPPER(username) FROM Users;
SUBSTRING() 提取字符串的子串。 SELECT SUBSTRING(‘hello world’, 1, 5); (返回 ‘hello’)
REPLACE() 替换字符串中的子串。 SELECT REPLACE(‘hello world’, ‘world’, ‘MySQL’); (返回 ‘hello MySQL’)
TRIM() 去除字符串两端的空格。 SELECT TRIM(‘ hello ‘); (返回 ‘hello’)

3.数值函数 (Numeric Functions)

用于对数值数据执行数学运算。

函数 描述 示例
ROUND() 对数字进行四舍五入到指定的小数位数。 SELECT ROUND(123.456, 2); (返回 123.46)
CEIL() / CEILING() 向上取整，返回大于或等于参数的最小整数。 SELECT CEIL(4.1); (返回 5)
FLOOR() 向下取整，返回小于或等于参数的最大整数。 SELECT FLOOR(4.9); (返回 4)
ABS() 返回一个数的绝对值。 SELECT ABS(-100); (返回 100)
MOD() 返回两数相除的余数。 SELECT MOD(10, 3); (返回 1)

4.日期与时间函数

函数 描述 示例
NOW() 返回当前的日期和时间。 SELECT NOW(); (例如: ‘2025-09-09 10:55:00’)
CURDATE() 返回当前日期。 SELECT CURDATE(); (例如: ‘2025-09-09’)
CURTIME() 返回当前时间。 SELECT CURTIME(); (例如: ‘10:55:00’)
YEAR() / MONTH() / DAY() 从日期中提取年、月、日。 SELECT YEAR(‘2025-01-20’); (返回 2025)
DATE_FORMAT() 格式化日期。 SELECT DATE_FORMAT(NOW(), ‘%Y年%m月%d日’); (返回 ‘2025年09月09日’)
DATEDIFF() 计算两个日期之间的天数差。 SELECT DATEDIFF(‘2025-10-01’, ‘2025-09-01’); (返回 30)

5.控制流函数 (Control Flow Functions)

允许在SQL查询中实现 if-then-else 逻辑。

函数 描述 示例
CASE SQL中的 if-else 语句，非常强大和常用。 见下方示例
IF() CASE 的简化版，用于简单的二元判断。 SELECT IF(score >= 60, ‘及格’, ‘不及格’) FROM Grades;
IFNULL() / COALESCE() 如果第一个表达式为NULL，则返回第二个表达式的值。COALESCE 可以接受多个参数。 SELECT IFNULL(nickname, ‘无昵称’) FROM Users;
