排序算法

发表于 2026-01-17 更新于 2026-07-16
本文字数： 2k 阅读时长 ≈ 2 分钟

今天集中攻克了 7 道高频面试题。重点总结了 LRU 的 O(1) 实现细节、链表归并排序的断链技巧，以及回文串的中心扩散法。但本次的主题是再记忆排序算法，做一个排序算法的归类和理解。

一、核心算法速查表

算法名称 平均时间 最坏时间 最好时间 空间复杂度 稳定性 备注 (面试金句)
冒泡排序 $O(n^2)$ $O(n^2)$ $O(n)$ $O(1)$ 稳定 教学意义大于实战，但需知道如何优化（Flag标志位）。
选择排序 $O(n^2)$ $O(n^2)$ $O(n^2)$ $O(1)$ 不稳定 无论数据好坏都是 $O(n^2)$，因为交换次数少，写操作敏感时可用。
插入排序 $O(n^2)$ $O(n^2)$ $O(n)$ $O(1)$ 稳定 数据量小或基本有序时，效率极高（甚至优于快排）。
希尔排序 $O(n^{1.3})$ $O(n^2)$ $O(n)$ $O(1)$ 不稳定 插入排序的改进版，打破了 $O(n^2)$ 的魔咒，但现代库函数较少用。
归并排序 $O(n \log n)$ $O(n \log n)$ $O(n \log n)$ $O(n)$ 稳定 由于稳定且性能稳定，常用于对象排序（如 Java 的 Arrays.sort 对对象用归并）。
快速排序 $O(n \log n)$ $O(n^2)$ $O(n \log n)$ $O(\log n)$ 不稳定 综合性能之王。最坏情况可通过随机化 Pivot 避免。
堆排序 $O(n \log n)$ $O(n \log n)$ $O(n \log n)$ $O(1)$ 不稳定 空间复杂度 $O(1)$ 的最强王者。常用于 Top K 问题。
计数排序 $O(n+k)$ $O(n+k)$ $O(n+k)$ $O(n+k)$ 稳定 牺牲空间换时间，仅适用于范围较小的整数。

1.1算法的稳定性

稳定：冒泡排序、插入排序、归并排序、基数排序（冒插归基稳定）

不稳定：快速排序，希尔排序，选择排序，堆排序（快希选堆不稳定）

二、算法原理解释

1.冒泡排序

核心逻辑：

  1. 从头开始，两两比较，如果前一个比后一个大，就交换。
  2. 一轮扫描结束后，最大值一定被移到了最后一位（就位）。
  3. 重复上述过程，直到没有元素需要交换。

2. 选择排序

核心逻辑：

  1. 在未排序序列中找到最小元素，存放到排序序列的起始位置。
  2. 再从剩余未排序元素中继续寻找最小元素，然后放到已排序序列的末尾。

3.插入排序

核心逻辑：

  1. 将数组分为“已排序区间”和“未排序区间”。
  2. 每次从未排序区间取出一个元素，在已排序区间从后往前扫描。
  3. 如果已排序的元素比它大，就向后移动，直到找到合适的位置插入。

4.快速排序

核心逻辑：

  1. 选基准：随便挑一个数（通常是第一个、最后一个或中间）作为 Pivot。
  2. 分区 (Partition)：把比 Pivot 小的扔左边，比 Pivot 大的扔右边。这一步做完，Pivot 就在它最终应该在的位置上了（这是快排最关键的性质）。
  3. 递归：对左右两边继续重复上述过程。

5.归并排序

核心逻辑：

  1. **分 (Divide)**：递归地把当前序列平均分割成两半，直到子序列长度为 1（由一个元素组成的序列天然有序）。
  2. **治 (Conquer)**：将两个有序的子序列合并成一个有序序列（双指针法：谁小取谁，放入新数组）。

面试价值：它是唯一一个稳定且时间复杂度永远是 $O(n \log n)$ 的算法，常用于对象排序或外部排序（内存装不下时）。

6.堆排序

核心逻辑：

  1. 建堆：把数组构建成一个大顶堆（根节点最大）。
  2. 交换：把堆顶（最大值）和数组最后一个元素交换（最大值归位）。
  3. **堆化 (Heapify)**：把剩下的 $n-1$ 个元素重新调整为大顶堆。
  4. 重复步骤 2-3。

面试价值：不需要额外空间 ($O(1)$)，但因为内存跳跃访问（索引 $i$ 的孩子在 $2i$），缓存命中率不如快排，所以实战中比快排慢。

7.希尔排序

核心逻辑：

插入排序在处理大规模乱序数据时很慢（因为一次只能移动一位）。

希尔排序先让数组中间隔为 Gap 的元素进行插入排序（宏观调控）。

不断缩小 Gap，直到 Gap=1，此时数组已经基本有序，最后做一次插入排序即可。

8. 基数排序

一句话原理：非比较排序。按位（个位、十位、百位…）进行分配和收集。

核心逻辑：

  1. 准备 10 个桶（0-9）。
  2. 先按个位数字，把元素扔进对应的桶里，然后按顺序倒出来（此时个位有序）。
  3. 再按十位数字，把元素扔进桶里，倒出来。
  4. 重复直到最高位。

要求：必须从低位到高位（LSD），且每一轮入桶必须使用稳定的排序算法（通常是计数排序的变种），否则高位的排序会打乱低位的顺序。
