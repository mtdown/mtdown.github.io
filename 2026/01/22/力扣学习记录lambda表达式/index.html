2.Comparator 万能口诀

根据学习，永远是写（a,b）根据目标效果，选择是a-b还是b-a

目标效果 核心写法 记忆口诀
小数值在堆顶 / 排前面 a - b 前减后（我减你）
大数值在堆顶 / 排前面 b - a 后减前（你减我）
String 长度短在堆顶 a.length() - b.length() 前减后
String 字典序靠前 a.compareTo(b) 顺着比

而对于一些复杂的情况，我们可以在里面增加代码串。逻辑是一样的，返回的数值小于0，就是选择数对的A上浮

3.扩展用法

不止是这个小根堆，大根堆场景，我们还能用在别的地方：

3.1 给数组/列表排序 (Arrays.sort / Collections.sort)

3.2 给二维数组排序

3.3 自动排序的 Map 和 Set (TreeMap / TreeSet)

这两个数据结构和 PriorityQueue 是亲兄弟。

  • PriorityQueue：只保证堆顶是有序的。
  • TreeMap / TreeSet：保证内部所有元素永远是有序的。

4. Java Stream 流操作 (.max, .min, .sorted)

Java 8 的流式编程极其依赖这个写法。

场景：在一堆对象里找“最…”的那一个

假设你有一堆 Student 对象，想找分数最高的那个。

5.总结

场景 代码片段 作用
优先队列 new PriorityQueue<>((a, b) -> a - b) 决定谁在堆顶
数组排序 Arrays.sort(arr, (a, b) -> a - b) 决定谁排前面
列表排序 list.sort((a, b) -> a - b) 决定谁排前面
Tree结构 new TreeMap<>((a, b) -> a - b) 决定Key的存储顺序
流操作 stream.max((a, b) -> a - b) 决定谁算最大值
